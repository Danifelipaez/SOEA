# Acceptance Criteria

## Purpose
Define the conditions that must be met for SOEA to be considered ready for the pilot deployment
and, later, for institutional rollout. Coordinators and project owners use these criteria to
validate the system before sign-off.

## Scope
Pilot acceptance criteria and post-pilot readiness criteria.

---

## Pilot Acceptance Criteria

A schedule generated by SOEA for the pilot dataset is accepted when ALL of the following are true:

### Schedule Quality

| # | Criterion | How to Verify |
|---|---|---|
| AC-01 | Zero hard constraint violations in the generated schedule | `summary.hardConstraintViolations = 0` in JSON output |
| AC-02 | Soft constraint fitness score is within 20% of the theoretical minimum | Compare against baseline score from manual schedule |
| AC-03 | Every session in the pilot dataset appears exactly once in the output | Count sessions in output = count sessions in input |
| AC-04 | All instructor availability rules are respected | Cross-check output against instructor availability file |
| AC-05 | No physical space exceeds its capacity | Verify `enrolledStudents ≤ space.capacity` for all in-person sessions |
| AC-06 | Alternancia rules are correctly applied (Type A and B sharing a space) | Manual spot-check by coordinator |

### System Performance

| # | Criterion | How to Verify |
|---|---|---|
| AC-07 | Optimization pipeline completes within 10 minutes for pilot data | Measure wall-clock time in logs |
| AC-08 | The API returns a valid JSON response (matches `json-output-spec.md`) | Schema validation test |

### Usability

| # | Criterion | How to Verify |
|---|---|---|
| AC-09 | At least 2 academic coordinators have reviewed the schedule and approved it | Coordinator sign-off record |
| AC-10 | The schedule can be exported and read without additional tools | Open JSON file in a standard viewer |

## Post-Pilot Readiness Criteria

Before institution-wide rollout:

| # | Criterion |
|---|---|
| AC-P01 | The system handles the full semester dataset (all programs) within performance targets |
| AC-P02 | Role-based access control is verified for all four roles (Admin, Coordinator, Instructor, Student) |
| AC-P03 | Data ingestion accepts real institutional Excel files without errors |
| AC-P04 | The system is deployed to a staging environment and tested by IT |
| AC-P05 | A user guide or onboarding document exists for Coordinators |
